XML
 21. github.com, click + --> New repository - cоздать внешний репозиторий c названием XML.
 22. git clone (ссылка на репозиторий) - клонировать репозиторий XML на локальный компьютер.
 23. cd XML / cat > new.xml / ctrl+c - Внутри локального XML создать файл “new.xml”.
 24. git add "new.xml" - добавить файл под гит.
 25. git commit -m "new.xml" - закоммитить файл.
 26. git push - отправить файл на внешний GitHub репозиторий.
 27. vim "new.xml" / insert / esc / :wq - отредактировать содержание файла “new.xml” - написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате XML.
 28. git add "new.xml" / git commit -m "new.xml" / git push - отправить изменения на внешний репозиторий.
 29. cat > preferences.xml / ctrl+c - создать файл preferences.xml
 30. vim "preferences.xml" / insert / esc / :wq - в файл preferences.xml добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате XML.
 31. vim "skills.xml" / insert / esc / :wq - создать файл sklls.xml добавить информацию о скиллах которые будут изучены на курсе в формате XML
 32. git commit -am "preferences/skills" - cделать коммит в одну строку.
 33. git push - отправить сразу 2 файла на внешний репозиторий.
 34. Add new file/Create new file - на веб интерфейсе создать файл bug_report.xml.
 35. Commit new file - сделать Commit changes (сохранить) изменения на веб интерфейсе.
 36. Edit this file - на веб интерфейсе модифицировать файл bug_report.xml, добавить баг репорт в формате XML.
 37. Commit changes - сделать Commit changes (сохранить) изменения на веб интерфейсе.
 38. git pull - синхронизировать внешний и локальный репозиторий XML
